How To Graffiti Code


In This Tutorial

In this tutorial you will learn basic skills for using Graffiti Code. If you have experience with other drawing languages such as PostScript, ActionScript or Processing, then some of what is described here will be familiar to you. But even so, the syntax and environment are different and there are other more subtle differences between Graffiti Code and other similar tools, so there is much to be learned from this tutorial.

Some of the skills you will learn here are:

  • Preparing a drawing surface
  • Creating and positioning shapes
  • Coloring shapes
  • Defining new words
  • Signing in
  • Archiving code
  • Sharing code

But first, a word on the motivation to draw with code.

Why Code?

When we draw with code, we calculate shapes and colors rather than copy them from what we see. We draw with code because we want to create using processes and values that can be manipulated by a machine rather than by our hand. Using a machine in this way allows us to articulate form using language that fits how we think. The properties that are important to us can be made explicit. We have language for controlling shape, size and position. We can directly re-size, re-position, re-shape, and re-color by re-phrasing their description. We can repeat with variation any and all forms our code describes.

Code gives us a notation for recording and archiving our ideas in a way that transcends the here and now. Code written at one time and place for one purpose can be remixed in other times and places for other purposes. It is this indirection between idea and form that makes drawing with code so powerful and full of potential meaning. Sol LeWitt relates how this separation of idea and form is important to the conceptual artist. He writes: "When an artist uses a conceptual form of art, it means that all of the planning and decisions are made beforehand and the execution is a perfunctory affair. The idea becomes a machine that makes the art."(Paragraphs on Conceptual Art, Sol LeWitt, Artforum, September 1967). Whether we are making conceptual art or designing a button for a web page, writing code allows us to think about the essence of the idea without having to think about all its possible realizations. With code we capture all that is common to a set of values without having to know what is different about them.

My hope is that by the end of this tutorial you will begin to appreciate how code can be a machine that makes the art.

NOTE: Graffiti Code programs consist of sequences of expressions. The expressions we will be using in this section are called function applications. You may remember from high school math class that a function maps a set of values to another set of values. The Graffiti Code functions that interest us most in this tutorial are ones that return values representing shapes that get drawn on the digital surface that we configure with other functions.

Graffiti Code provides two functions that affect the drawing surface. They are size and background. Creating and positioning shapes


Graffiti Code provides functions for constructing and manipulating shape values. Shape constructors include:

  • curve
  • ellipse
  • line
  • path
  • point
  • rectangle
  • text
  • triangle

Shape modifiers include:

  • translate
  • color
  • fill
  • stroke
  • rotate
  • skewx
  • skewy
  • scale
  • font-size

In this section we will be focusing on the ellipse constructor to demonstrate creating shapes and the translate modifier to demonstrate positioning shapes.

Coloring shapes


Graffiti Code provides primitive functions for computing colors and applying those colors to shape values. These functions include:

  • rgb red green blue - computes an opaque color value
  • rgba red green blue alpha - computes a transparent color value
  • fill color shape - sets fill color
  • stroke color shape - sets stroke color
  • color color shape - sets stroke and fill color

Coloring shapes

We apply the coloring functions to a color and a shape whose color is to be updated. In addition to the 256 gray values described in the discussion of the background function, we can compute colors using RGB values using the rgb and rgba functions. The rgb function is applied to three numeric values between 0 and 255 representing the red, green and blue components of a color. rgba is applied to four values, the first three being the same as those for rgb and the last one being a numeric value between 0 and 100 representing the transparency of the computed color where 0 represents complete transparency and 100 represents complete opacity.

Using the stroke transparency of 0 is an idiom for representing that we want no stroke color. We use 0 0 0 (black) for simplicity; all completely transparent colors appear the same.

At this point you're probably thinking to yourself, "Wow! There's a lot of redundancy in this code. Why can't I just say 'stroke rgba 0 0 0 0' and 'ellipse 50 50' once?" and "These so called expressions are pretty long and gangly. This isn't going to be fun." Not to worry. The solution involves defining new words that wrap all that redundancy into neat little functional packages that are configured just the way we want. In addition to providing tidier abstractions for the things you want to represent, these functions avoid the pitfalls that come with using global state to manage shared properties.

Defining new words

Perhaps one of the most interesting features of Graffiti Code is it's use of what computer scientists call functional programming to describe the computation of form. In Graffiti Code, we can define new words that map to functions that compute form however we want.

Defining functions is simple for simple functions, but can become quite elaborate and hence powerful. Signing in


Signing in to Graffiti Code allows you to save your work to the archive and to share your work as Github Gists.

Here is were we will describe how to sign in. (You can probably figure it out by clicking the link named "Sign In" in the Graffiti Code header.

Archiving code


Once you've signed into Graffiti Code you can save your work to the archive. All archived pieces are publically viewable and sharable. But most importantly they are stored for you and others to use as the starting point for new pieces.

In the Workspace view, click the Archive button to save the current code. Code that has not been significantly modified will not be saved. Significant changes involve changing the compiled meaning of the code. For example, whitespace changes do not constitute significant modifications.

Sharing code with Github


NOTE: This feature has not yet been implemented.

Once you've signed into Graffiti Code you can share your code as a Github Gist.

In the Workspace panel, click the Share button to save the current source and object code as a Gist. Code that has not been significantly modified since the last time it was shared, will not be saved. Significant changes involve changing the compiled meaning of the code. For example, whitespace changes do not constitute significant modifications. Once your code has been shared on Github, the URL to its Gist will be recorded as a comment in the code.
