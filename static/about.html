About Graffiti Code


Graffiti Code is an experiment in drawing with code. It consists primarily of a workspace, a language and an archive.

The workspace includes an editor and a viewer. When the code in the editor is valid language, it is interpreted and the drawing in the viewer is updated.

The language of Graffiti Code is functional and statically typed with minimal syntax. References to user definitions (keyword 'let') are folded at authoring time to produce a graph that consists of expressions of primitive values and operations. This graph is interpreted to generate a drawing.

The archive displays saved drawings. Saved drawings can be edited or viewed in a new browser window. Editing an existing drawing creates a fork of the original. Each archived drawing is an HTML document with an unique URL.

Click around the archive to see what has been done. If you feel lucky, login with Persona and try writing some code yourself. Beware though, the only help we currently offer is what you can gleen from the archive. Oh, and we recommend using a recent version of Firefox. (Something to do with a bug in Webkit while positioning LaTeX objects embedded in SVG, in case you are wondering.)
